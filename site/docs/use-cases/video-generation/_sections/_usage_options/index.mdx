#### Video Generation Configuration

You can adjust several parameters to control the video generation process, including dimensions and the number of inference steps:

<LanguageTabs>
    <TabItemPython>
        ```python
        import openvino_genai as ov_genai

        pipe = ov_genai.Text2VideoPipeline(model_path, "CPU")
        fps = 25
        video = pipe.generate(
          prompt,
          # highlight-start
          width=512,
          height=512,
          num_videos_per_prompt=1,
          num_inference_steps=30,
          num_frames=161,
          guidance_scale=7.5,
          frame_rate=fps
          # highlight-end
        ).video

        save_video("genai_video.avi", video, fps)
        ```
    </TabItemPython>
    <TabItemCpp>
        ```cpp
        #include "openvino/genai/video_generation/text2video_pipeline.hpp"
        #include "imwrite_video.hpp"

        int main(int argc, char* argv[]) {
            const std::string models_path = argv[1], prompt = argv[2];

            ov::genai::Text2VideoPipeline pipe(models_path, "CPU");
            float fps = 25;
            ov::Tensor video = pipe.generate(
                prompt,
                // highlight-start
                ov::genai::width(512),
                ov::genai::height(512),
                ov::genai::num_videos_per_prompt(1),
                ov::genai::num_inference_steps(30),
                ov::genai::num_frames(161),
                ov::genai::guidance_scale(7.5f),
                ov::genai::frame_rate(fps)
                // highlight-end
            ).video;

            imwrite_video("genai_video.avi", video, fps);
        }
        ```
    </TabItemCpp>
</LanguageTabs>

:::info Understanding Video Generation Parameters

- `negative_prompt`: Negative prompt for video(s) generation.
- `width`: The width of resulting video(s).
- `height`: The height of resulting video(s).
- `num_frames`: Number of frames to generate for each video. Higher values produce longer videos but increase compute and memory.
- `num_videos_per_prompt`: Specifies how many video variations to generate in a single request for the same prompt.
- `num_inference_steps`: Defines denoising iteration count. Higher values increase quality and generation time, lower values generate faster with less detail.
- `guidance_scale`: Balances prompt adherence vs. creativity. Higher values follow prompt more strictly, lower values allow more creative freedom.
- `generator`: Controls randomness for reproducible results. Same generator seed produces identical videos across runs.
- `frame_rate`: Target video FPS used by the pipeline.

For the full list of generation parameters, refer to the [Video Generation Config API](https://docs.openvino.ai/2026/api/genai_api/_autosummary/openvino_genai.VideoGenerationConfig.html).

:::

### Decode Latents Manually (Advanced)

`Text2VideoPipeline` exposes a `decode()` method that converts a raw latent tensor into video frames. This is useful when you capture the final latent via a callback and want to decode it with a specific `decode_timestep`.

:::note
`decode_timestep` only has an effect for models whose VAE was trained with timestep conditioning, such as **LTX-Video 0.9.1** (`Lightricks/LTX-Video-0.9.1`). For the base LTX-Video model this parameter is ignored and defaults to `0.0`.
The value should be the last scheduler timestep normalized to `[0, 1]`: `timestep / max_timestep`. The pipeline computes this automatically during `generate()`, but you can supply it explicitly when calling `decode()` directly.
:::

<LanguageTabs>
    <TabItemPython>
        ```python
        import openvino_genai as ov_genai

        pipe = ov_genai.Text2VideoPipeline(model_path, "CPU")

        final_latent = None

        def callback(step, num_steps, latent):
            global final_latent
            final_latent = latent
            return False

        result = pipe.generate(prompt, callback=callback)

        # Decode with an explicit decode_timestep for LTX-Video 0.9.1+
        result = pipe.decode(final_latent, decode_timestep=0.05)
        video = result.video
        ```
    </TabItemPython>
    <TabItemCpp>
        ```cpp
        #include "openvino/genai/video_generation/text2video_pipeline.hpp"

        int main(int argc, char* argv[]) {
            const std::string models_path = argv[1], prompt = argv[2];

            ov::genai::Text2VideoPipeline pipe(models_path, "CPU");

            ov::Tensor final_latent;
            auto callback = [&final_latent](size_t step, size_t num_steps, ov::Tensor& latent) -> bool {
                final_latent = latent;
                return false;
            };

            pipe.generate(prompt, ov::genai::callback(callback));

            // Decode with an explicit decode_timestep for LTX-Video 0.9.1+
            auto result = pipe.decode(final_latent, /* decode_timestep= */ 0.05f);
            ov::Tensor video = result.video;
        }
        ```
    </TabItemCpp>
</LanguageTabs>
